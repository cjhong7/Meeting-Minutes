## 핵심 결정사항 7원칙 (모든 후속 설계의 기준)

1. **운영 주체 = 교사 개인 / 정적 사이트 / 백엔드 없음**
2. **AI = BYOK 전용(GPT-4o / Gemini / Claude 택1) + 로컬 시뮬레이션 폴백** (공용 데모 백엔드 폐기)
3. **데이터 = 100% 로컬 + 사용자 선택 시만 외부 저장**
4. **STT = Whisper + 브라우저 SpeechRecognition 듀얼**
5. **모드 = 단일 모드 (4가지 중 1개 선택, 혼용 X)**
6. **참석자 = 8명까지 1행, 9~16명 2행**
7. **파일 = HWPX + PDF + DOCX (구형 HWP 제외)**

## AI 엔진 정책 (모드 공통 — 단일 기준)

> 모든 모드가 공유하는 AI 엔진 처리 기준. 세부 프롬프트·후처리는 Ⅵ절에서 확장.

### 1. 지원 엔진 (BYOK, 주 엔진 1개 선택)
- **GPT-4o (OpenAI)** / **Gemini (Google)** / **Claude (Anthropic)** / 키 없음(로컬 시뮬레이션)
- 세 엔진 모두 브라우저 직접 호출 가능 (백엔드 없음 원칙 부합)
  - OpenAI: 브라우저 직접 호출 지원
  - Gemini: API 키 방식 브라우저 호출 지원
  - Claude: `anthropic-dangerous-direct-browser-access: true` 헤더로 브라우저 직접 호출

### 2. 엔진 선택 UI ([AI 설정 ⚙])
```
┌ AI 엔진 설정 ────────────────────────────┐
│ 사용할 엔진:                              │
│  ○ GPT-4o (OpenAI)    [키 입력_________]  │
│  ○ Gemini (Google)    [키 입력_________]  │
│  ○ Claude (Anthropic) [키 입력_________]  │
│  ○ 키 없음 (로컬 시뮬레이션)              │
│ ※ Claude 선택 시 녹음 음성변환은          │
│   브라우저 음성인식으로 대체됩니다.        │
└──────────────────────────────────────────┘
```
- 키는 crypto.subtle + 사용자 PIN으로 암호화해 IndexedDB에만 저장

### 3. ★ 엔진별 기능 차이 (중요)
| 기능 | GPT-4o | Gemini | Claude |
|---|---|---|---|
| 협의록 텍스트 생성 | ✓ | ✓ | ✓ |
| 음성→텍스트(STT) | ✓ Whisper | ✓ 오디오 직접입력 | ✗ → 브라우저 SR 대체 |
| 펜 손글씨 OCR(vision) | ✓ | ✓ | ✓ |
| 스캔 PDF OCR(vision) | ✓ | ✓ | ✓ |

### 4. 라우팅 규칙
- 주 엔진 1개 선택 → 협의록 텍스트·OCR은 선택 엔진으로 처리
- 음성 STT만 엔진 능력에 맞춰 자동: OpenAI=Whisper / Gemini=오디오 입력 / Claude=브라우저 음성인식
- 키 없음(시뮬레이션) 모드: AI 호출 없이 양식만 채움, 음성·OCR은 브라우저 무료 기능만

## Ⅰ. 사업 개요 — 확정

### 1. 사업명
**학교 협의록 작성기** (가칭) / 코드명 `anti-conver`

### 2. 목적
- 학교 현장의 다양한 회의 환경(녹음·계획서·타자·펜)에 모두 대응하는 **단일 협의록 작성 도구** 구축
- 교육기관 표준 양식(개조식, 가/나/다 계층)을 **AI가 자동 생성**
- 회의 원본 자료를 **사용자 기기에 안전하게 보관**, 인쇄·엑셀로 출력

### 3. 필요성
- 협의록 작성에 회의당 30분~1시간 소요 → 교사 업무 가중
- 학교마다 양식·문체가 제각각 → 표준화 필요
- 기존 도구는 회의 유형별로 분산(녹음앱·문서편집기·OCR앱) → 통합 필요
- AI 도구 등장으로 자동화 가능해졌으나, **학교용 양식·문체·민감정보 보호**를 모두 만족하는 도구 부재

### 4. 운영·배포 모델
- **개발 주체**: 교사 개인
- **배포 방식**: 정적 웹사이트 (GitHub Pages 등 무료 호스팅), URL만 알면 누구나 접속
- **사용자 AI 비용**: 학교/교사가 본인 OpenAI 또는 Gemini 키 입력해 사용(BYOK)
- **서버 인프라**: 없음 (백엔드 0, 데이터베이스 0, 사용자 데이터는 사용자 기기에만)
- **앱 형태**: 다운로드 앱이 아닌 **웹 접속** (설치 없음, PWA로 홈화면 추가는 가능)
- **면책 고지문**: 키 노출·데이터 손실·AI 오류는 개발자 비책임. 첫 화면에 명시.

### 5. 범위

**포함**
- 4가지 입력 모드(녹음/계획서/타자/펜) → AI 협의록 생성
- 회의 메타데이터 입력(회의명·일시·장소·안건·참석자 3~16명)
- 미리보기, 인쇄, 엑셀 다운로드
- 로컬 보관함, 선택적 클라우드 백업(Google Drive / OneDrive)

**제외**
- 다중 사용자 동시 편집(협업 기능)
- 회의 일정 관리·알림
- 학교 간 데이터 공유·통계 집계
- 구형 HWP 파일(HWPX·PDF·DOCX로 우회)
- 네이티브 앱(웹앱으로만)

### 6. 대상 사용자
- 초·중·고등학교 교사
- 평균 ICT 활용 수준 가정 (엑셀·한글·브라우저 사용 가능)

## Ⅱ. 사용자 요구사항 — 확정

### 1. 전역(모든 화면 공통)
- **상단 도구바**: [새로작성] [내 기기에 저장] [보관함 열기] [엑셀 다운로드]
- '엑셀 다운로드'는 협의록 생성 후에만 활성화
- '내 기기에 저장'은 사용자가 저장 위치 선택: **내 컴퓨터 폴더 / 태블릿 폴더 / Google Drive·OneDrive 클라우드**
- 백그라운드 보관함(IndexedDB) 자동 저장은 별도로 항상 동작

### 2. 1쪽 (메타+모드 선택)
- **회의 정보**: 회의명, 일시(날짜+시간 분리 입력), 장소
- **안건**: 항목별 입력 + [+안건 추가] 버튼으로 다건 안건 등록
- **참석자**: 3~16명 가변 칸 (8명까지 1행, 9~16명 2행)
- **작성 방식 선택**: 녹음 / 계획서 / 타자 / 펜 (라디오)
- 모드 선택 후 그 모드의 입력 UI는 1쪽 하단에 펼쳐짐

### 3. 2쪽 (협의록 미리보기)
- 학교 표준 양식 그대로 표시 (테이블 + 본문)
- 1쪽 메타 자동 반영
- AI 생성 협의록 본문 (개조식, 가/나/다 계층)
- **읽기 전용** — 수정은 엑셀 다운로드 후 엑셀에서 작업
- 데스크톱 좌우 분할 / 태블릿·모바일 탭 전환

### 4. 4가지 입력 모드별 핵심 요구

| 모드 | 핵심 동작 |
|---|---|
| 녹음 | 마이크 연결 확인(권한+입력 레벨 시각화) → [녹음 시작] → 녹음 중 상태 표시 → 정지 → 녹음 파일 다운로드 → [협의록 작성]. 3시간 무중단 보장. |
| 계획서 | HWPX/PDF/DOCX 업로드 → 텍스트 추출 미리보기(편집 가능) → [협의록 작성] |
| 타자 | 텍스트 입력란(자동 저장 5초마다) → [협의록 작성] |
| 펜 | 펜·지우개 도구 → 캔버스 필기 → [텍스트 추출] (GPT-4o vision, BYOK 키 필요) → 추출 텍스트 표시(편집 가능) → [협의록 작성]. 키 없으면 수동 입력 폴백. |

### 5. 출력
- **인쇄**: 학교 표준 양식 그대로
- **엑셀**: 가로 1쪽 가득(여백 조정), 세로는 내용에 따라 자동 확장

### 6. 원본 자료 보관
- 회의 1건 = 메타 + 원본(녹음/계획서/펜 이미지) + 협의록 묶음
- 협의록과 함께 자동 보관

### 7. 비기능 요구사항
- 언어: 한국어 단일
- 브라우저: Chrome 최신 2개 버전 우선
- 오프라인: PWA 캐시 후 펜·타자·로컬시뮬레이션 사용 가능 (Whisper·AI 호출은 인터넷 필수)
- AI 응답: 보통 10~30초 — 진행 상태 표시
- 저장 한도: IndexedDB 약 500MB → 1시간 녹음 회의 기준 약 8~16건

### 8. 보강 결정사항
- 보관함 정렬: 최신 회의 순
- 보관함 검색: 회의명·일시·안건 키워드
- 회의 자동 삭제: 없음 (사용자 명시 삭제만)
- 협의록 재생성: 가능, **AI 호출 직전 매번 비용 확인 모달** ("약 50~150원 추가 로드됨. 계속?")
- 다크모드: 불필요 (인쇄 양식 기준 흰 바탕)
- 다국어: 불필요

### 백엔드를 뺀 이유 (재참조용)
1. 운영 주체가 교사 개인 — 24/7 가용성·보안·호스팅비 부담 불가
2. AI를 BYOK로 결정 — 키 숨김용 프록시 불필요
3. 데이터는 100% 로컬 — 통로 자체가 보안·법적 책임의 원천
4. 파일 지원 HWPX+PDF+DOCX — 모두 브라우저 단독 파싱 가능
5. 학교망 환경 — 정적 사이트는 한 번 캐시되면 망 정책 영향 적음

### 3. 참석자 가변 칸 동작

| 인원 | 배치 | 칸 폭 |
|---|---|---|
| 3명 | 1행 × 3칸 | (W - 간격×2) / 3 |
| 8명 | 1행 × 8칸 | (W - 간격×7) / 8 |
| 9명 | 2행 (5+4) | (W - 간격×4) / 5 |
| 16명 | 2행 (8+8) | (W - 간격×7) / 8 |

- 인원 수는 드롭다운(3~16) 조정
- 줄일 때 입력된 이름 있으면 삭제 확인 모달
- 인쇄·엑셀에서도 동일한 행/칸 비율 유지

### Ⅴ-1. 녹음 모드 — 확정

#### 가. 사용자 흐름
```
[1] 녹음 준비
  ├─ STT 방식 선택 (◉ Whisper / ○ 브라우저)
  ├─ 마이크 연결 확인 → 입력 레벨 막대 시각 확인
  └─ (Whisper 선택+키 없음) 경고: "키 없으면 브라우저 STT만 가능"
[2] [녹음 시작]
  ├─ Wake Lock 활성화 (화면 꺼짐·절전 방지) + 안내 배너
  ├─ beforeunload 경고 등록
  ├─ 타이머 + 파형/레벨 표시
  ├─ 10분 세그먼트 단위 분할 저장 → Web Worker로 MP3 인코딩 → OPFS
  └─ 1분마다 IndexedDB 임시 저장 (크래시 복원)
[3] 녹음 중
  ├─ "● 녹음 중 00:42:13" + 레벨 막대
  ├─ (브라우저 STT 시) 실시간 텍스트 누적
  ├─ 무음 5분 지속 시 알림
  └─ 3시간 도달 시 자동 정지
[4] [녹음 정지]
  ├─ 세그먼트 확정 → 하나의 회의 녹음으로 묶음
  └─ Wake Lock·beforeunload 해제 → STT 변환 시작
[5] STT 변환
  ├─ Whisper: MP3 세그먼트 순차 전송 → 전문 합산 (진행률)
  ├─ 브라우저 STT: 누적 텍스트 사용
  └─ 변환 텍스트 표시 → 편집 가능
[6] [녹음 파일 다운로드](선택) + [협의록 생성](Ⅵ절 공통)
```

#### 나. 마이크 연결 확인 (2단계)
- **권한**: `getUserMedia({audio})` → "연결됨 ✓" / "거부됨 ✗"
- **입력 레벨**: Web Audio `AnalyserNode` 실시간 데시벨 → 막대 `▮▮▮▮▮▯▯▯`가 말하면 움직임
- 막대 안 움직이면 "마이크에 소리가 들어오지 않습니다" 경고

#### 다. 3시간 무중단 보장
| 위협 | 대응 |
|---|---|
| 메모리 폭증 | 10분 세그먼트 분할 → OPFS 즉시 기록 후 메모리 비움 |
| 크래시 | 1분마다 IndexedDB 저장 → 재접속 시 "이전 녹음 복원" 모달 |
| 화면 꺼짐·절전 | Wake Lock 자동 ON + 안내 배너 |
| 실수로 탭 닫기 | beforeunload 경고 |
| 백그라운드 throttle | 경고 배너: "녹음 중 이 탭을 화면에 두세요" |

#### 라. ★ 백엔드 없는 Whisper 청킹 (핵심 기술)
- Whisper 25MB 한도 + 백엔드 없음 → **처음부터 10분마다 완전한 파일로 분할 녹음**
- 각 10분 MP3(약 5~10MB)는 독립 재생·전송 가능 → Whisper 순차 전송
- timeslice 조각을 그냥 자르면 헤더 없어 깨짐 → 반드시 세그먼트 = 완전한 파일
- 다운로드 시 세그먼트를 이어 붙여 단일 MP3 제공

#### 마. MP3 변환 처리 (포맷 결정 반영)
- 녹음 → 10분 세그먼트 → **Web Worker에서 즉시 MP3 인코딩(lamejs)** → OPFS
- 녹음 중 직전 세그먼트를 백그라운드 변환 → UI 멈춤 없음
- MP3 세그먼트를 Whisper에도 그대로 전송 → 변환 결과물 1개로 다운로드·STT 동시 충족
- [녹음 정지] 시점엔 대부분 변환 완료 → 다운로드 거의 즉시

#### 바. STT 듀얼 모드
| | Whisper API | 브라우저 SpeechRecognition |
|---|---|---|
| 정확도 | 높음 | 중간 |
| 비용 | 분당 약 8원(BYOK) | 무료 |
| 실시간 표시 | ✗ | ✓ |
| 3시간 무중단 | ✓ | △ (자동 재시작, 경계 단어 누락 위험) |
| 키 필요 | OpenAI 키 | 불필요 |

- **기본값: Whisper.** 브라우저 STT는 키 없음/오프라인/짧은 회의용 폴백
- 브라우저 STT 끊김은 `onend`/`onerror`에서 지수 백오프(300ms→1s→3s) 재시작

#### 사. 예외 처리 / 폴백
| 상황 | 처리 |
|---|---|
| 마이크 권한 거부 | 버튼 비활성 + "자물쇠 아이콘 → 마이크 허용" 안내 |
| 녹음 중 마이크 분리 | 감지 → 알림 + 그때까지 분량 저장 |
| Whisper 변환 실패 | 재시도 버튼 + "브라우저 STT로 시도" 폴백 + 원본 보존 |
| 일부 세그먼트 실패 | 성공분 합산 + `[N구간 변환 실패 — 재시도]` 표시 + 구간별 재시도 |
| 저장 공간 부족 | 시작 전 용량 경고 ("3시간 ≈ 약 180MB 필요") |

#### 아. 녹음 파일 첨부 (외부 기기 녹음 지원)
스마트폰 음성메모·별도 녹음기로 녹음한 파일을 업로드해 STT→협의록 생성. 브라우저 녹음 불안정 시 백업 경로.

- **UI**: 녹음 모드 안에서 `[🎤 실시간 녹음] / [📎 파일 첨부]` 탭 전환
- **흐름**:
  ```
  [1] 드래그앤드롭/클릭 선택 → 지원 포맷 안내
  [2] 검증: 포맷 확인 + 길이·용량 표시("48분/42MB") + 미리듣기 플레이어
  [3] STT 변환
       ├─ 25MB 이하 → Whisper 그대로 전송
       ├─ 25MB 초과 → ffmpeg.wasm으로 시간 단위 자동 분할 후 순차 전송
       └─ 진행률 표시
  [4] 변환 텍스트 표시 → 편집 → [협의록 생성]
  ```
- **지원 포맷**: MP3 · M4A · WAV · WebM · OGG · MP4(오디오) (모두 Whisper 호환)
- **큰 파일 분할**: ffmpeg.wasm 자동 분할 (3시간 파일도 브라우저 안에서 안정 처리). 첫 로드 시 약 30MB 다운로드, PWA 캐시로 1회만. 백엔드 없음 원칙 부합.
- **STT 제약**: 첨부 파일은 Whisper만 가능(브라우저 SR은 마이크 실시간 전용). 키 없으면 "직접 텍스트 입력" 폴백
- **원본 보관**: 첨부 파일도 회의 1건에 함께 보관(Ⅱ-6 정책)
- **부수 효과**: ffmpeg.wasm 도입 → Ⅸ. 기술 스택 절에 의존성·로딩 전략 반영 필요

#### 자. 확정 요약
- 포맷 MP3 / 3시간 자동 정지 / 부분 실패는 성공분 합산+구간 표시 / Wake Lock 자동 ON / 세그먼트 10분 / 1분 자동저장
- 입력 경로 2종: 실시간 녹음 + 외부 파일 첨부(ffmpeg.wasm 자동 분할)

### Ⅴ-2. 계획서 모드 — 확정

계획서를 업로드해 텍스트를 추출하고, 그 내용을 근거로 "계획안"(AI 프롬프트가 녹음/타자와 다름, Ⅵ절)을 생성하는 모드.

#### 가. 사용자 흐름
```
[1] 파일 업로드 (드래그앤드롭/클릭) — 여러 개 동시 허용
      └─ 지원: .hwpx .pdf .docx
[2] 텍스트 추출 (브라우저 단독, 백엔드 없음)
      ├─ PDF  → pdf.js (텍스트 PDF). 스캔 PDF면 GPT-4o vision OCR 자동
      ├─ DOCX → mammoth.js
      └─ HWPX → JSZip 압축해제 + section XML 파싱
[3] 추출 결과 미리보기 (편집 가능)
      ├─ 파싱 오류·깨진 글자 직접 수정
      └─ 안건 자동 제안 (1쪽 안건 칸에 "추가 제안"으로, 기존 입력 미덮어씀)
[4] [협의록 생성] → Ⅵ절 AI 사양(계획안 모드)
```

#### 나. 포맷별 추출 방식
| 포맷 | 라이브러리 | 비고 |
|---|---|---|
| PDF | pdf.js | 텍스트 PDF만 직접 추출. 스캔(이미지) PDF는 GPT-4o vision OCR 자동 시도 |
| DOCX | mammoth.js | 표·서식 일부 손실, 본문 안정 추출 |
| HWPX | JSZip + XML 파싱 | `Contents/section*.xml`의 `<hp:t>` 텍스트 수집. 전용 파서 빈약 → 폴백 필수 |

#### 다. 스캔 PDF OCR (결정 반영)
- 텍스트가 없는 PDF면 pdf.js로 각 페이지를 canvas 이미지로 렌더링 → GPT-4o vision 전송 → 텍스트 추출
- BYOK 키 재사용, 페이지당 약 30원. 키 없으면 "텍스트 PDF로 다시 저장하거나 직접 붙여넣기" 안내 폴백

#### 라. HWPX 파싱 폴백 (불확실성 대응)
- 1차: JSZip으로 section XML 추출 → 텍스트 노드 수집
- 실패 시: "이 HWPX를 읽지 못했습니다. 한글에서 텍스트 복사 후 붙여넣거나 PDF로 저장해 다시 업로드" 안내
- 텍스트 직접 붙여넣기 입력란 항상 제공

#### 마. 추출·구조 처리 (결정 반영)
- 다중 파일: 본 계획서+첨부 등 여러 개 업로드 → 추출 텍스트를 이어붙여 AI 전달
- 안건 자동 제안: 제목·번호 항목(1., 가., ○ 등) 감지 → 1쪽 안건 칸에 추가 제안(체크 채택), 기존 입력 미덮어씀
- 표: 행을 줄바꿈, 셀을 공백/탭으로 평탄화
- 파일 크기: 20MB·100페이지 초과 시 경고

#### 바. 확정 요약
- HWPX/PDF/DOCX, 여러 파일 허용 / 스캔 PDF는 GPT-4o vision OCR / 안건 자동 제안(추가형) / 추출 텍스트 편집 가능 / HWPX 실패 시 붙여넣기 폴백

### Ⅴ-3. 타자 모드 — 확정

현장에서 직접 친 회의 내용을 근거로 협의록(회의록 모드)을 생성.

#### 가. 사용자 흐름
```
[1] 회의 내용 입력 (큰 textarea, 자동 높이 확장)
      ├─ 5초 디바운스 IndexedDB 자동 저장 (새로고침·이탈 복원)
      ├─ 글자 수 실시간 표시
      ├─ 말머리 도우미: [협의안건]/[전달사항] 버튼으로 빠른 삽입
      └─ 🎤 음성 받아쓰기 보조 (브라우저 SpeechRecognition → textarea)
[2] [협의록 생성] → Ⅵ절 AI 사양(회의록 모드)
```

#### 나. 확정사항
- 입력 구조: **자유 입력(단일 textarea) + 말머리 도우미**
- 음성 받아쓰기 보조: **제공**(🎤 버튼, 브라우저 음성인식, 녹음 모드와 별개의 단순 입력 보조)
- 자동 저장: 5초 디바운스 + beforeunload 경고 + 복원 모달
- 글자 수: 실시간 표시, 약 2만 자 초과 시 "AI 비용·시간 증가" 안내

### Ⅴ-4. 펜 모드 — 확정

태블릿 스타일러스 필기를 OCR로 텍스트화 후 협의록(회의록 모드) 생성. OCR은 선택 엔진의 vision 사용(GPT-4o/Gemini/Claude 모두 가능).

#### 가. 사용자 흐름
```
[1] 단일 캔버스에 필기 (세로 스크롤로 공간 확보 가능)
      ├─ 도구: ✏ 펜 / 🧽 지우개 / ─ 굵기 / 🎨 색 / 🗑 전체 지우기 / Undo·Redo
      └─ 포인터 이벤트(마우스·터치·스타일러스 통합)
[2] [텍스트 추출]
      ├─ 캔버스를 PNG(2x)로 렌더링 → 선택 엔진 vision 전송 → 텍스트
      └─ 키 없으면 "직접 입력" 폴백
[3] 추출 텍스트 미리보기 (편집 가능, OCR 정확도 한계 안내)
[4] [협의록 생성] → Ⅵ절 AI 사양(회의록 모드)
```

#### 나. 입력 기술
- 포인터 이벤트로 마우스·터치·스타일러스 통합
- 필압(`pressure`) 지원 기기에서 선 굵기 자연스럽게
- 손바닥 오터치 방지(`pointerType==='pen'`만) 옵션
- 캔버스 디바이스 픽셀비 2x → 인쇄·OCR 선명도 확보

#### 다. 손글씨 OCR 정확도 (기대치 정렬)
- 한국어 손글씨는 인쇄체 대비 정확도 낮음(흘림체·악필 특히)
- "OCR 결과는 정확하지 않을 수 있으니 확인 후 수정하세요" 강조
- 또박또박 쓰기 팁 제공

#### 라. 확정사항
- 손글씨 원본: PNG로 회의 1건에 보관(보관용)
- **협의록 출력(인쇄·엑셀)에는 OCR 텍스트만 사용**, 원본 이미지는 미포함
- 캔버스: **한 페이지(단일 캔버스)**, 세로 스크롤로 공간 확보
- 도구: 펜/지우개/굵기/색/전체지우기/Undo·Redo
- 키 없을 때: 직접 입력 폴백

## Ⅶ. 데이터·보안 모델 — 확정

### 1. 저장 위치 (전부 사용자 브라우저, 개발자 서버 전송 0)
| 데이터 | 저장소 |
|---|---|
| 회의 메타·협의록 텍스트·참석자 | IndexedDB |
| 녹음 MP3 세그먼트·펜 PNG·계획서 원본 | OPFS |
| API 키 | IndexedDB(암호화) |
| 임시 녹음 진행분 | IndexedDB(1분 단위, 크래시 복원) |

### 2. 외부로 나가는 데이터는 단 두 경우 (둘 다 사용자→AI사 직접, 개발자 미경유)
- ① AI 협의록 생성 시 입력 텍스트 → OpenAI/Gemini/Claude
- ② 음성 STT 시 오디오 → Whisper/Gemini
- 키 없음(시뮬레이션) 모드는 외부 전송 0

### 3. API 키 보호 (확정)
- 평문 저장 금지. `crypto.subtle`(AES-GCM) 암호화 후 IndexedDB
- **PIN 선택 설정** + 설정 시 세션당 1회 입력 (공용 PC는 PIN 권장, 개인 PC는 생략 가능)
- PIN은 PBKDF2로 암호화 키 파생. **PIN 분실 시 키 복구 불가 → 재입력 안내**
- 세션 종료 시 복호화된 키를 메모리에서 제거

### 4. 클라우드 백업 (확정: 동기화 폴더 활용)
- **OAuth 직접 연동 안 함** (앱 등록·검증·100명 제한 함정 회피)
- 대신 PC의 **Google Drive/OneDrive 데스크톱 동기화 폴더에 저장** → 자동 클라우드 업로드
- 앱은 File System Access API로 "저장 폴더 선택"만 → OAuth·백엔드 불필요, 범용 배포 최적

### 5. 로컬 파일 저장 기술
- File System Access API (Chrome·Edge): 폴더 선택→직접 쓰기, 같은 폴더 재저장
- 미지원(Firefox·Safari): 일반 다운로드 폴백
- 형식: 회의 1건 = JSON(메타+협의록) + 원본 파일 묶음(ZIP)

### 6. 외부 전송 동의 & 면책 (확정)
- **비용 확인 모달에 "입력 내용이 OpenAI/Gemini/Claude로 전송됩니다" 통합**, 첫 1회 동의 후엔 비용 모달만 유지
- 첫 실행 면책 고지: "교사 개인 무료 제공. 데이터 손실·키 노출·AI 오류 비책임. 학생 개인정보 입력 주의"

### 7. 데이터 관리

- 저장 용량 사용량 표시

## Ⅷ. 출력 — 인쇄·엑셀 — 확정

### 1. 세 출력 양식 통일
- 2쪽 미리보기(HTML) = 인쇄(print CSS) = 엑셀(ExcelJS) 동일 데이터·레이아웃
- 미리보기가 곧 인쇄·엑셀 모습

### 2. 엑셀 양식 구조 (학교 표준 재현)
```
┌─────┬──────────────────────────────────────────┐
│         ○○학교 협의록 (제목, 셀 병합)            │
├─────┼──────────────────────────────────────────┤
│회의명│ (병합)                                    │
├─────┼──────────────────┬─────┬─────────────────┤
│일 시│                  │장 소│                 │
├─────┼──────────────────┴─────┴─────────────────┤
│참석자│ 이름+서명빈칸 8칸/행 (9~16명은 2행)        │
├─────┼──────────────────────────────────────────┤
│안 건│ 1. ... 2. ...                             │
├─────┴──────────────────────────────────────────┤
│ ■ 회의록 ... (2026. 5. 27.)                     │
│ [결정 및 논의 사항]  1. / 가. / -               │
│ [전달 사항]  1. ...                             │
└──────────────────────────────────────────────────┘
```

### 3. "가로 1쪽 가득 / 세로 가변" (요구사항 직결)
- ExcelJS: `fitToPage:true`, **`fitToWidth:1`**(가로 1쪽 압축), **`fitToHeight:0`**(세로 무제한 확장)
- 용지 A4 세로(`paperSize:9`, `orientation:'portrait'`), 여백 최소화로 가로 폭 최대 활용
- 본문 셀 `wrapText:true` + 행 높이 자동

### 4. 인쇄 (print CSS)
- 좌측 입력 패널 숨김, 2쪽 미리보기만
- `@page { size:A4 portrait; margin:최소 }`
- 표 행 분리 방지 `page-break-inside:avoid`

### 5. 참석자 배치 (확정: 이름+서명란)
- 각 참석자 칸 = **이름 + 서명·날인 빈칸** 2단
- 8명까지 1행, 9~16명 2행, 셀 병합으로 칸 폭 균등
- 화면·인쇄·엑셀 동일 비율

### 6. 여러 쪽 처리 (확정)
- 상단 양식(회의명·일시·참석자)은 **첫 쪽에만**, 2쪽부터는 협의록 본문만 이어짐

### 7. 기타 확정
- 페이지 방향: A4 세로 고정
- 펜 손글씨 이미지: 출력 미포함, OCR 텍스트만(Ⅴ-4)
- 글꼴: 맑은 고딕

> 확정: 순수 JS(빌드 없음) + 로컬 호스팅 라이브러리 + AI는 fetch 직접 + ffmpeg lazy load

### 1. 라이브러리 의존성
| 용도 | 라이브러리 | 무게 | 로딩 |
|---|---|---|---|

- 오프라인에서 펜·타자·로컬 시뮬레이션·보관함 동작(AI·STT만 인터넷 필요)

### 5. 확정사항

- 라이브러리 제공: **로컬 호스팅** (오프라인 PWA·학교망 차단 안전)

### 4. 비용 절감 포인트
- 짧은 회의·정리된 내용은 타자+저가형이면 거의 무료 수준
- 녹음 길면 무료 브라우저 STT 선택 가능(정확도↓ 감수)
- BYOK라 각 AI사 대시보드에서 월 사용 한도(spend limit) 직접 설정 가능

### 5. 사용자 핵심 답변(재확인)
- 검색엔진 토큰: 사용 안 함(요약이라 LLM 토큰만)
- **비용 부담 주체 = 키를 만든 사람**(BYOK):
  - 개발자(도구 만든 교사 본인) 부담 = 0 (서버 없음)
  - AI 사용 비용은 "쓰는 측"이 부담 → 두 방식 중 선택
    - A. 학교 공용 키(권장): 학교가 예산/법인카드로 키 1개 발급, 교사들이 공유 → 교사 개인 부담 0, 단 행정 절차 필요
    - B. 교사 개인 키: 교사 본인 카드로 발급 → 개인 부담
  - 앱은 부담 주체를 강제하지 않음. 각 학교 행정 결정 사항
- 키 없으면 완전 무료(시뮬레이션)

### 6. 부가 기능 (확정)
- **사용량·예상 비용 추정 표시**: AI 응답의 토큰 수를 로컬 누적 → "이 기기에서 이번 달 약 N원(추정)" 표시. 한계 명시(기기별 집계만, 정확한 청구는 AI사 대시보드). 외부 전송 없음
- **비용 절감 가이드**: [AI 설정]에 "짧은 회의는 타자+저가형, 긴 녹음은 비용↑, 완전 무료는 키 없음 모드" 안내

## Ⅺ. 일정·우선순위 — 확정

> 전제: 순수 JS·백엔드 없음·1인(교사) 개발, 방과후·주말 짬짬이(주 5~10시간) → 기간 넉넉히

### 1. 전략 (확정): 공통 기반 먼저 + 4모드 병렬
- "4개 모드 동시 개발"을 채택하되, 미완성 장기화를 막기 위해 **공통 기반을 먼저 깔고 그 위에 4모드를 병렬로 얹음**
- 공통 기반은 어느 모드든 반드시 필요 → 낭비 최소

### 2. 개발 단계
| 단계 | 내용 | 우선순위 | 예상(짬짬이) |
|---|---|---|---|
| 0 뼈대 | index.html, 1쪽/2쪽 레이아웃, 상태 모델, 반응형 | 🔴 필수 | 1~2주 |
| 1 메타+양식 | 회의명·일시·장소·안건·참석자(8+8) + 2쪽 미리보기 | 🔴 필수 | 1주 |
| 2 AI 엔진 | 엔진 라우팅·프롬프트·후처리 + 키 없음 시뮬레이션 | 🔴 필수(공통 검증) | 2주 |
| 3 출력 | 인쇄 CSS + 엑셀(가로1쪽/세로가변) | 🔴 필수 | 1~2주 |
| 4 저장·보안 | IndexedDB 보관함 + 키 암호화(PIN) + 폴더 저장 | 🔴 필수 | 2주 |
| 5 4모드 병렬 | 타자·계획서·녹음·펜을 공통 기반 위에 병렬 구현 | 🟡 중요 | 6~10주 |
| 6 다듬기 | 클라우드(동기화폴더)·PWA·사용량 표시·절감 가이드 | 🟢 개선 | 2주 |

※ 5단계 내부 난이도: 타자(쉬움) < 계획서 < 펜 < 녹음(3시간 무중단, 최고 난이도)

### 3. MVP 정의
- 단계 0~4 + 타자 모드 = **타자만으로 협의록 생성→출력→저장이 되는 완전한 앱**
- 이걸로 실제 회의에 써보며 AI 품질·양식 검증 후 나머지 모드 확장

### 4. 현실성 메모
- 짬짬이 + 4모드 병렬은 완성까지 오래 걸림 → 공통 기반 완성 시점에 한 번 실사용 점검 권장
- 녹음 모드는 난이도 최고라 가장 마지막에 안정화

## Ⅻ. 위험요소·대응 — 확정

### 1. 기술적 위험
| 위험 | 영향 | 대응 |
|---|---|---|
| 3시간 녹음 중 크래시 | 회의 손실 | 1분마다 IndexedDB 임시저장 + 10분 세그먼트 + 복원 모달 |
| 절전·백그라운드로 녹음 중단 | 녹음 끊김 | Wake Lock 자동 + 경고 배너 + beforeunload |

| 손글씨 OCR 정확도 낮음 | 협의록 부정확 | "확인·편집 필수" 강조 + 또박또박 안내 |
| ffmpeg.wasm 로딩 실패 | 분할 불가 | "파일 나눠 첨부" 폴백 |
| AI 양식 미준수 | 출력 깨짐 | 후처리 정규화(기호 제거·어미 보정) |
| AI 환각 | 허위 협의록 | "추측 금지" 프롬프트 + 빈 섹션 비움 + 사용자 검토 |

### 2. 운영·환경 위험
| 위험 | 영향 | 대응 |
|---|---|---|
| 학교망 AI API 차단 | 생성 불가 | 키없음 시뮬·펜·타자 동작 + 다른 망 시도 안내 |
| 브라우저 호환(FF·Safari) | 일부 기능 미동작 | Chrome·Edge 권장 + 다운로드 폴백 |
| 키 발급 진입장벽 | 시작 못 함 | 단계별 가이드 + 키없음 모드 우선 체험 |

### 3. 보안·법적 위험
| 위험 | 영향 | 대응 |
|---|---|---|
| 공용 PC 키 유출 | 키 도용 | crypto.subtle 암호화 + PIN + 사용 후 삭제 안내 |
| 학생 개인정보 AI 전송 | 개인정보 이슈 | 전송 동의 모달 + 민감정보 주의 고지 + 익명화 권고 |

- 녹음 동의: 녹음 시작 전 1회 안내 팝업
- 면책 고지: 최초 1회 동의 + [설정]에서 상시 열람
